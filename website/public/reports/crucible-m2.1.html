What the crucible caught

A full audit of M0–M2 — then this gate proving the fixes hold

After M2 shipped, a read-only audit swept the M0–M2 code on five axes: adequacy to spec, code quality, attack / algorithmic / resilience security, and multi-tenant isolation. The tenant-isolation read boundary (the non-forgeable namespace= mediation) and the crypto core came back sound. But it found one critical data-integrity defect in the backup fan-out and three high-severity correctness / security gaps. All were fixed, re-run through the offline gate (unit + envtest + lint, all green), and then re-validated here on real Ceph — first pass, no iteration — including a purpose-built regression the standard seed could never trigger.

critical · C1

Cross-namespace mover name collision

A cluster-DR run fans out one child Backup of the same name into every namespace, and its per-PVC objects share the operator namespace. Named only from (backup, pvc), two namespaces with a same-named PVC (data, …) collided — the second silently adopted the first’s Job, its own volume never backed up. Names are now namespace-qualified. The seed uses distinct PVC names, so a bespoke same-name-in-two-namespaces spec was added — it passes here on real Ceph.

high · H1

Mutable repository identity & mode

A location’s clusterID/s3.* and mode had no immutability guard — editing them silently re-points the location at a different repository (orphaning every backup) or downgrades WORM. Pinned with update-only CEL, with an envtest.

e0b3e1e

high · H2 / H3

Lock-strip race & cross-location GC

The mover⇄unlock quiescence check and Job create weren’t atomic, so a stale-lock unlock --remove-all could strip a live mover’s lock (create-then-verify closes it). And discovery’s projection GC ran cluster-wide, so two locations deleted each other’s projections (now location-scoped).

why it matters

Green offline ≠ correct

The critical collision compiled, linted and passed the full envtest + kind e2e suites — latent only because the seed happens to use distinct PVC names per namespace. A field-shaped regression on real storage is what pins it.
