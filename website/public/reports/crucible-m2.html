  • Partial-restores ONLY the selected file (R7) 41.7s

    A file-level include restores exactly the one selected file; every other file in the PVC is left untouched.

  • Recreate-restores to an EXACT match (extras removed) 31s

    Recreate (restic restore --delete) reconciles the volume to the snapshot exactly — the seeded files match and the local extra is gone.

  • Fails CLOSED when a tampered projection points at another namespace’s run (R14) 6s

    A forged cluster-origin projection claiming another namespace’s run is caught fail-closed by defense in depth — discovery prunes the fake (no matching repository snapshots), the mediated namespace= filter backs it up — and never borrows the foreign snapshot.

  • Restores the repo coordinate into a freshly-created namespace, PVC sized from the snapshot tags, data byte-identical 1m38.2s

    The exit criterion: a deleted namespace is reconstituted from the repository coordinate alone — the target namespace and its PVC are recreated (sized from the pvcsize/pvcclass/pvcmodes snapshot tags) via the pvc-transplant handover, and the data comes back byte-for-byte.

What the crucible caught

Two product defects only real Ceph could surface — then fixed

envtest uses a fake mover, enforces no RBAC, and has no pvc-protection controller; kind’s admin client hides missing grants. So the offline suites — which passed clean — could not see the real transplant handover on a cluster where a finished pod actually pins a claim. The first full live run came back 24 · 4 · 3; each failure was diagnosed against restic (or apiserver) ground truth and fixed, converging on the clean run above.

product · #1

Transplant handover deadlock

After the restore mover completes, its pod kept the staging claim pinned by the pvc-protection finalizer — but the pvc-transplant handover must delete that claim to re-point the PV. The result is now stamped on the Job and the pod deleted each pass (scoped pods:delete in the operator namespace); the Job stays as the durable “mover ran” marker.

product · #2

Duplicate restore plans

A repository holding several snapshots of one PVC under a run (a re-used run name) made the namespaced restore build one plan per entry and restore the PVC twice. restorableVolumes now dedupes by PVC, matching the ClusterRestore path.

test harness · #3

Admission-aware & self-referential seed

The M2 VAPs (correctly) reject a user-created cluster-origin Backup and a selector-less ClusterBackup; the seed’s MANIFEST.sha256 listed itself. Fixtures made admission-aware, the tamper authored as the operator, the manifest excludes itself.
